# Mate Codex Handoff R25

> 이번 라운드: 카드 생성 실패 메시지를 구체화.

## 배경

- 카드 생성 실패 시 화면에 `카드 내용을 확인해주세요.`만 보여 사용자가 어떤 항목을 고쳐야 하는지 알 수 없었다.
- 실제 원인은 Zod 검증 실패였지만 API가 세부 이유를 버리고 있었다.

## 변경 사항

- `lib/cards/schema.ts`
  - 카드 생성 스키마의 각 필드에 한국어 검증 메시지를 추가했다.
  - 제목, 장소, 모집인원, 설명, 마감시간/모임시간 순서 등을 구체적으로 설명한다.
- `app/api/cards/route.ts`
  - `INVALID_CARD_INPUT` 발생 시 첫 번째 Zod issue 메시지를 응답으로 내려준다.
- `app/cards/new/NewCardForm.tsx`
  - 날짜 변환 후 모임 일시가 과거인지, 신청 마감시간이 모임 이후인지 제출 전에 안내한다.

## 예시 메시지

- `제목은 4자 이상 입력해주세요.`
- `설명은 10자 이상 입력해주세요.`
- `모임 일시는 현재보다 이후여야 합니다.`
- `신청 마감시간은 모임 일시보다 이전이어야 합니다.`
