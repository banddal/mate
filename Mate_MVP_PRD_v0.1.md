# Mate MVP PRD — v0.1 (Lean)

> 혼자 하기 아쉬운 순간에, 함께할 사람을 찾는 가장 가벼운 방법.
> 작성일: 2026-07-06 · 상태: 설계 착수용 · 원칙: 가설 1이 증명되기 전의 모든 기능은 부채다.

---

## 1. 제품 정의

**Mate(mate)는** 시간·장소·활동이 정해진 상황에, 일회성 mate 팀을 만들어주는 서비스다.
만남이 끝나면 관계는 플랫폼에 축적되지 않는다. 남는 것은 "약속을 지켰는가"뿐이다.

**Mate가 아닌 것**
- 데이팅 앱이 아니다 → 사람 프로필을 브라우징하는 화면이 존재하지 않는다
- 내 히스토리에는 내가 진행한 만남만 기록이 남는다
- 커뮤니티 앱이 아니다 → 지속 참여·멤버십·모임방을 만들지 않는다
- SNS가 아니다 → 팔로우·DM·피드 축적이 없다
- 만일 무엇인가 확인할 수 있다면 어떤 만남을 주선했고 참여했는가 까지다

**정체성 리트머스** (모든 기능 결정에 적용)
> 메인 화면이 사람이면 데이팅, 모임방이면 커뮤니티, **상황(이벤트) 카드면 Mate다.**

## 2. 핵심 가설 (V0.1이 검증하는 것)

**가설 1 (생사)** — 사람들은 상황 카드를 보고, 낯선 사람과의 오프라인 만남에 실제로 나오는가?

**가설 2 (정체성)** — 소멸 UX를 경험한 유저가 다시 돌아오는가?
→ 소멸이 "부담 제거"로 작동하면 성립, "애착 차단"으로 작동하면 피벗 신호.

이 두 개 외의 모든 것(수익, 배지, 구독, 결제)은 V0.2 이후의 질문이다.

## 3. 제품 원칙 5

### P1. 상황이 먼저다
메인 화면은 상황 카드 리스트. 카드의 시각 위계: **① 무엇을·언제·어디서 ② 호스트가 건 것 ③ 사람 정보(최소)**.
프로필 사진은 카드에 노출하지 않는다 — 매칭과정에서 일부 공개. (외모 브라우징·타겟팅 최소화)
유일하게 상대의 프로필을 볼 수 있는 건 내가 내건 미팅에 경쟁률이 생긴 경우에 한한다
이마저도 모두 공개하지 않고 평점 지수, 미팅 성사 지수 등 간접 정보와 블러처리된 프로필 화면으로 간접적으로 확인할 수 있다. 

### P2. 카드는 활동을 말한다
욕망은 유입되어도 좋다. 그러나 카드가 공시하는 것은 항상 **구체적 활동**이다.
- 적극 허용: 제주 흑돼지 / 야구 직관 / 전시 / 판교 금요 맥주
- 조건부 허용: 술·야간·여행지 등 활동은 구체적이나 안전 리스크가 높은 카드
- 금지: 감정 상태·취약성 공시 카드  
  예: 실연 / 우울 / 외로움 / 위로해줄 사람 / 오늘 같이 있어줄 사람
  실연한 사람은 들어와도 된다./하지만 “실연 메이트”라는 카드는 열면 안 된다. 대신 “금요일 밤 포차 mate”에 참여하게 해야 한다.

### P3. 조건은 상황의 잉여만
호스트가 제시할 수 있는 것 = **상황에 이미 묶여 있는 비용** (티켓·예약·좌석·공간).
걸 수 없는 것 = 현금 사례비, 참석 보상, 개인정보 대가, 보상 암시. 기타 사회미풍양속을 거스르는 등의 이유로 법률적 책임의 대상이 되는 경우 
→ 이 한 줄이 서비스의 종(種)을 가른다. costly signal 필터의 핵심.

### P4. 소멸은 유저 화면의 규칙, 로그는 백엔드의 규칙
- 유저 화면: 모임 종료 → 채팅방 소멸, 프로필 재접근 불가, 팔로우/DM 없음. 다시 만나려면 그 자리에서 직접.
- 백엔드: 법정 최소 로그 보존, 신고 발생 건은 동결 보존. **협상 불가 전제.**
- 표면 언어: 미팅이 사라지는 것은 이 서비스의 중요 속성. "은밀·익명·흔적 없는" 금지. 어휘는 **가벼움** — "부담 없이, 딱 오늘만."
- 소멸은 마이크로카피로 존재: *"모임이 끝나면 서로를 찾을 수 없어요. 다시 만나고 싶은 사람에게는, 그 자리에서 직접 물어보세요."*

### P5. 남는 것은 약속 이행뿐
후기 = 사실 체크 4개
- 실제로 참석했는가
- 시간을 지켰는가
- 카드 설명과 실제 진행이 일치했는가
- 이 상황 카드가 다시 열리면 참여할 의향이 있는가
외모·매력·호감·인기 점수 없음. 점수가 아니라 완료 사실(배지)의 축적(야구. 전시, 여행, 향후 술자리 등).

## 5. 카테고리 구조 — 알리바이 경사(gradient)

금지가 아니라 경사로 통제한다. 그레이 카드를 열고 싶을수록 건전한 카드에서 이력을 먼저 쌓게 된다.

| Level | 카테고리 | 개설/참여 조건 |
|---|---|---|
| **L1 강한 알리바이** | 직관·공연·전시·페스티벌 (시간·장소가 외부 확정) | 기본 인증만. 마찰 최소 |
| **L2 중간** | 맛집·카페·러닝·퇴근 후 맥주 | 3인 이상 기본. 완료 이력 1회 이상 |
| **L3 약한 알리바이** | 술·야간·여행지 (제주·부산) | 완료 이력 N회 + **계정 생성 후 N주 경과** + 상대측 완료 확인 필수(파밍 방어). 소그룹 강제, 성인 인증, 모니터링 강화 |

⚠️ **초기 L3 공백은 버그가 아니라 설계다.** 성비 붕괴 방지의 유일한 순서. 성장 압박으로 임계치를 낮추자는 논의가 오면 이 줄을 다시 읽을 것.

## 6. 성공 지표 (8주)

**North Star** — 첫 mate 완료 + 소멸 경험 후 **30일 내 재신청/재생성 비율 ≥ 25%**

| 지표 | 기준 | 검증하는 것 |
|---|---|---|
| 자발 생성 카드 비율 | ≥ 50% | 운영자 데모가 아닌 진짜 공급 |
| 승인 → 실제 참석률 | ≥ 75% | 가설 1 |
| 여성 참여 완료율 (카테고리별) | 추적 | 성비 붕괴 조기 경보 |
| L2→L3 도달 유저 발생 | 존재 여부 | 경사 구조 작동 |
| 신고율 | ≤ 3% | 안전 바닥 |

❌ "모임 50건" 류의 볼륨 지표는 성공 기준이 아니다 — 운영자가 만들 수 있는 숫자는 데모지 검증이 아니다.

## 7. 결정 로그 (채택/기각 — 재론 방지용)

| 제안 | 판정 | 사유 |
|---|---|---|
| 보증금: 게스트 노쇼분 → 호스트 보상 | ✅ 채택 (V0.2) | 플랫폼이 노쇼로 수익 얻는 구조 방지. 단 상호 체크인 없이는 보상 로직 사기 벡터 → 결제와 함께 V0.2 |
| 완료 사실(배지) + 만족도 평가 | 🔧 수정 채택 | 만족도 좋은 이벤트를 많이 만든 경우 뱃지, 특정 분야 이벤트 많이 만든 경우 뱃지 |
| 배지 2개로 L3 해제 | 🔧 수정 채택 | 횟수만으론 파밍 가능. 시간 경과 + 만족도 결과 확인, 신뢰점수 획득(내부 시스템용) |
| 거절 → "정원 초과 소멸" 안내 | 🔧 수정 채택 | 가짜 사유는 다크패턴. 개별 통지 생략 + 마감 시 일괄 해소로 |
| 예매 캡처 필수 업로드 | ❌ 기각 | 위조 비용 제로 → 가짜 신뢰 제조. 병목(호스트 공급)에 마찰 추가. 진짜 검증은 V0.2 에스크로 결제 |
| 호스트 구독 ("이 호스트의 다음 카드 알림") | ❌ 기각 | 우회 팔로우 그래프 → 인플루언서화 → 포식자의 청중 축적 복원. 대체: 상황 템플릿 구독 (기능 #8) |
| "자극적 카드 바이럴 = 낮은 CAC" | ❌ 기각 | 역선택 구조: 싼 유입은 원치 않는 풀, 필요한 유저(여성·위험회피형)는 신뢰 신호로만 옴. 획득 채널 = 성비 설계의 일부 |
| 범죄 방치 / 개인정보 판매 / 증거 소멸 | ❌ 영구 기각 | 선별 효과 역작동(포식자 유인) + 운영자 형사 리스크. 실존적 위험은 보존, 물리적 면책은 제로 |

## 8. 결정 완료 (2026-07-06)

1. **첫 진지** — ✅ 잠실 직관 (L1). 사유: 실제 인맥·접근성 보유, 알리바이 최강으로 마찰 최소. 리스크: 프로야구 시즌 리듬에 종속 → 휴식기 대비 예비 카테고리(전시·공연) L1 내에서 확보해둘 것.
2. **4주 공급 시딩** — ✅ 지인 네트워크에 호스트 역할 요청. 직관 다니는 지인 5~10명을 개별 컨택해 "이번 주 직관 있으면 카드로 열어달라" 요청. ⚠️ §6 지표("자발 생성 카드 비율 ≥ 50%")를 훼손하지 않으려면, 지인이 **실제로 참석하는 진짜 모임**을 여는 것이어야 한다 — 운영자 대리 개설(가짜 수요)과 구분할 것.
3. 서비스명 — 보류 (가칭 Mate 유지, 상표·검색 확인은 빌드와 병행, 논블로킹)
