# Mate — 코덱스 핸드오프 (Round 7)

> 이전 라운드: `Mate_Codex_Handoff_R1~R6.md`. 이번 라운드는 새 기능이 아니라 **효율성 검토 후 대대적인 범위 축소**입니다.
> 상세 근거는 `Mate_Backend_Spec.md` §13(결정 로그) 참조. 이전 라운드에서 안내했던 내용 중 일부가 이번에 **취소/축소**되니 아래를 꼭 확인해주세요.
> ⚠️ 여전히 설계 단계입니다.

---

## 취소/축소된 것 (이전 핸드오프 내용 정정)

| 이전 안내 (R2~R6) | 변경 사항 |
|---|---|
| 호스트 출석확인(롤콜) UI (R2) | **V0.1에서 만들지 않는다** — L3 전용 장치였는데 L3 자체가 안 열림. 참여자 후기(review)의 `attended` 자기신고만으로 충분 |
| 실제 성인 인증(KYC) 온보딩 화면 (R2, R4) | 이미 보류였는데 이번에 확실히 취소 — L3 열릴 때 재논의 |
| L3 임계치 안내 UI, `l3.capacity_min/max`를 `GET /api/config`에서 받아오기 (R3) | L3 카드 생성 자체가 API에서 차단되므로, L3 카드 생성 폼을 아예 안 만들어도 됨(카테고리 선택지에 L3 자체를 노출하지 말 것) |
| `app_config` 기반 설정값 (R3) | **코드 상수로 전환** — `GET /api/config`는 그대로 있지만 이제 DB가 아니라 공유 상수 파일을 그대로 반환. 프론트는 여전히 이 값을 하드코딩하지 말고 API로 받아 쓰면 됨(변경 방식만 바뀜) |
| admin 역할별 화면 분기(moderator/superadmin) (R4) | **단일 admin 티어로 통합** — `/admin/*`에 역할별 화면 분기 로직 넣지 않아도 됨 |
| Web Push 3단계 재시도 관련 안내 (R3) | 프론트 영향 없음(서버 내부 로직), 그냥 1회 재시도로 단순화됐다는 점만 참고 |

---

## 그대로 유효한 것

- 로그인/OTP 화면, `/cards/new`, `/feed`, `/rooms/[id]` 등 R1~R6에서 안내한 나머지 화면 스펙은 전부 유효
- 신청자 목록 2계층 응답(경쟁 시에만 블러 프로필 노출) — 유효
- 카드 검수 큐, 신고 처리 상태 머신 — 유효

---

## 운영(Ops) 작업 갱신

- **Vercel Cron 설정** — `vercel.json`에 cron 항목 추가 필요 (마감 해소 1~5분, 알림 발송 1분, 룸 정리 일 1회). Supabase pg_cron/Edge Function 설정은 **불필요해짐**(이전에 안내했다면 취소)
- 나머지(카카오 앱 등록, SMS 벤더, VAPID 키)는 기존 안내 그대로 유효
