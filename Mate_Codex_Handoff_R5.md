## 이번 라운드에서 새로 생긴 것

카드 생성 시 서버가 3단계로 판정합니다:
1. **하드 금지어 매칭** → 카드 자체가 생성되지 않음(`error.code='CONTENT_BLOCKED'`)
2. **연성 플래그 매칭 / L3 카테고리 / 현금성 패턴** → `status='pending_review'`로 생성, 검수 통과 전까지 피드에 안 보임
3. **위 어느 것도 아님(대다수 L1/L2)** → 즉시 `status='open'`, 바로 피드 노출

## 프론트엔드 작업 (신규/변경)

| 항목 | 변경 내용 |
|---|---|
| `/cards/new` 제출 결과 처리 | 3가지 응답 케이스 분기 필요: (a) 성공+즉시공개 → 카드 상세로 이동, (b) `pending_review` → "검수 중입니다" 안내 화면(피드엔 안 보이지만 호스트 본인은 상태 확인 가능), (c) `CONTENT_BLOCKED` → "활동 중심으로 구체적으로 작성해주세요" 같은 **일반화된 안내 문구**만 표시(정확히 어떤 단어가 걸렸는지 알려주는 UI 절대 금지 — 필터 회피 방지) |
| `/cards/[id]` (호스트 본인 뷰) | `pending_review`/`rejected` 상태를 호스트에게는 보여주되, 다른 유저에게는 그 카드 자체가 존재하지 않는 것처럼 처리(피드 미노출 + 상세 페이지 직접 접근도 차단) |
| `/admin` 카드 검수 큐 | `GET /api/admin/cards/review-queue` 목록 UI, 승인/반려 버튼. 반려 시 사유 입력(자유 텍스트, 호스트에게는 일반화된 정해진 문구만 노출 — moderator가 쓴 원문을 그대로 유저에게 보여주지 말 것 |

## 운영(Ops) 작업 (신규/변경)

- `banned_words` 초기 시드 데이터 준비 — `emotional_vulnerability`(실연/우울/외로움 등, PRD P2 금지 목록) + `cash_compensation`(사례비/페이/용돈/보상 등) 최소 목록으로 시작, 운영하며 추가
